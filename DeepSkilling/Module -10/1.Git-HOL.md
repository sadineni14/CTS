# Git HOL (Hands-On Lab)

## Objective

* Learn basic Git commands.
* Configure Git user settings.
* Create and initialize a Git repository.
* Add files to the repository.
* Commit changes.
* Push and pull code from a remote repository.

---

## Prerequisites

* Git installed on the system.
* GitHub or GitLab account.
* Internet connection.

---

## Git Configuration

Configure username and email:

```bash
git config --global user.name "Your Name"
git config --global user.email "your_email@example.com"
```

Verify configuration:

```bash
git config --global --list
```

---

## Create a Git Repository

Create and initialize a repository:

```bash
mkdir GitDemo
cd GitDemo
git init
```

Verify repository:

```bash
git status
```

---

## Create a File

Create a file named `welcome.txt`:

```bash
echo "Welcome to version control" > welcome.txt
```

View file content:

```bash
type welcome.txt
```

---

## Add File to Staging Area

```bash
git add welcome.txt
```

Check status:

```bash
git status
```

---

## Commit Changes

```bash
git commit -m "Added welcome file"
```

View commit history:

```bash
git log --oneline
```

---

## Remote Repository

Add remote repository:

```bash
git remote add origin <repository-url>
```

View remote:

```bash
git remote -v
```

---

## Push Changes

```bash
git push -u origin master
```

or

```bash
git push -u origin main
```

---

## Pull Changes

```bash
git pull origin master
```

or

```bash
git pull origin main
```

---

## Common Git Commands

| Command       | Description              |
| ------------- | ------------------------ |
| git init      | Initialize repository    |
| git status    | Check repository status  |
| git add       | Stage files              |
| git commit    | Save changes             |
| git log       | View commit history      |
| git remote -v | View remote repositories |
| git push      | Upload changes           |
| git pull      | Download changes         |

---

## Conclusion

This Hands-On Lab demonstrated Git configuration, repository creation, file tracking, committing changes, and synchronizing code with a remote repository. These operations form the foundation of version control and collaborative software development.
